## ⚠️ THE NON-NEGOTIABLE RULE — "No Number Without a Paper Trail"

This rule overrides everything else. Every numeric value, rating, AED figure, percentage, or score that appears on screen MUST satisfy ONE of these three conditions:

1. **Verified** — sourced from a real, citable document (Aldar annual report, ADX disclosure, UAE Central Bank, ISO/COSO standard, client policy doc). The user can click "ⓘ" and see the citation + retrieval timestamp.
2. **Illustrative / Sample / Dummy** — explicitly labeled on screen as sample data with a tooltip explaining what real data replaces it post-integration.
3. **AI Hypothesis** — generated by Claude/AI, badged as "Pending human approval" until a real user approves it.

If a value is none of the above, **it does not ship**. No "around 50M", no "approximately 30%", no silent math, no fabricated baselines, no invented sensitivity coefficients.

### Operational checklist before every commit

For each numeric value, calculation, or chart in the patch:

| Check | Required |
|---|---|
| Source documented? | Citation OR explicit "sample" label |
| Formula visible (if computed)? | Yes, with clickable upstream inputs |
| Could user mistake for production data? | If yes → MVP/PREVIEW badge + dummy-data tooltip |
| AI-generated? | "AI Hypothesis — pending approval" badge |

If any answer is "no" or "unsure" → **flag to Badal, do not guess**.

### Why this rule exists

The previous version of this platform shipped fabricated AED figures and uncalibrated sensitivity coefficients. The client called it out and lost trust. This rule is the contract that rebuilds that trust. **Drift = engagement death.**
